1. Introduction

1.1  Audience Description

The SSA program was developed to notify a caregiver if their child was left behind in the car seat via smartphone Application and Commercial Off-The-Shelf (COTS) product integration. The intended audience of this document is the development team, management, customer and the user.

The experience level expected of the user can range from a beginner with no prior Android device usage experience, to an expert. It is expected that the user will know how to operate Android device and how to pair it via Bluetooth to the COTS in the car. It is also expected that the user/caretaker will be able to properly secure the child in the correctly installed car seat. No special training is required to use SSA system.

1.2 Applicability Statement. This document applies to the XXXX version of the SSA system which has been developed using Arduino IDE  and Android Studio.

1.2.1 Hardware Environment        

1.2.1.1 Mobile Device Requirements                                                                  

Screen:

  • at least 2.5 inches in physical diagonal size
  • 100 dpi or more density,
  • aspect ratio between 1.333 (4:3) and 1.779 (16:9)
  •  touch screen navigation

Performance:

  • at least 1 GHz processor,  256MB of RAM, and 20MB of available memory space.

Bluetooth:

  • Version 2.0 wireless technology or higher

1.2.1.2 Car Requirements

  • Properly installed age-appropriate car seat.
  • Vehicle cigarette lighter receptacle or USB.

1.2.2 Software Environment

  • Android Gingerbread 2.3.3 or higher

1.3 Purpose Statement. 

The purpose of this document is to instruct the users on how to effectively and safely use the SSA system. The SSA system was developed to prevent caregivers from accidentally forgetting their child in the car. It was not intended for purposefully leaving a child in the car which is against the law in most states.

2. Body of Document.

2.1 Body of Instructional Mode: Full Screen Alarm Notification Trigger.

2.1.1 Scope.  The scope of the following instruction is to provide the user with description of the main software function - The Full Screen Alarm Notification Trigger. It will cover all the actions to be taken by user to trigger Full Screen Alarm ( set by default) on the users Android device.

2.1.2 Materials. The user must have the car with correctly installed car-seat and the Android device with installed App in order for  SSA system to operate.

2.1.3 Preparations. The users device must be paired with a car seat Arduino COTS via Bluetooth in order for SSA system to operate properly. The Arduino COTS should be connected to the car via cigarette lighter receptacle or USB ( if one is available).

2.1.4 Cautions and Warnings. Do not intentionally leave your child in the car to test the system. The following Method description is to illustrate to the user the way main function of the SSA system operates and under what circumstances the phone notification will be triggered.

2.1.5 Method. 

2.1.5.1 User Actions

  • Initiates the SSA system by placing child in the car seat.
  • Starts the car and drives to the destination.
  • Parks the car, turns off the ignition and removes the keys.
  • User then leaves the car forgetting to remove the child from the car seat. WARNING : do not leave your child in the car intentionally.
  • When user leaves the 30 foot range of car seats COTS Bluetooth, the Android device sounds and displays full screen alarm.
  • As soon as the full screen alarm is triggered, User must return to the car and remove the child from the car seat.
  • As soon as the child is removed from the car seat - the Full Screen Alarm turns off.
